Vue重构CNode社区项目总结

Jul 9, 2018 | 七月

前言:

前几天用Vue全家桶重构了CNode社区, 因为它们提供API数据接口可调用。
之前写的印象笔记是自己mock的一份数据,所有的数据交互都是用过vuex操
作的数据来呈现的。通过村塾本地localstorage,可实现刷新还是呈现最新的数据。

技术栈:

vue + vuex + vue-router + axios + less + echarts

其中遇到的问题

  • 通过父组件向子组件传递数据 props 为对象类型的。 在子组件中展示传递过来的数据之后,父组件的数据发生变化,子组件视图没有改变的原因

这个是在父组件向子组件传递数据,name对象,子组件通过 props接收作为自己data中的数据,点击按钮触发 emit通知父组件更新状态。

  1. 如果父组件直接通过 this.name.name = '齐庆福',那么子组件的视图会得到更新。

  2. 如果在父组件中直接将 this.name的引用指向另外一个对象,那么子组件的视图不会得到更新。

如果是第二种情况还想让子组件的视图得到更新,那么不要把 props的数据定义在 data中,直接在模板中通过 this.name使用。

  • 通过 router-link进行路由跳转并且携带参数。

query和params传递参数的展现方式

通过this.$router.push()进行路由跳转

  • 在组件中 style 引入外部less文件, 把scoped的去掉，否则引入无效

  • 使用的markdown文本插件,用户再回复的时候,自动将文字转为带有div标签的html文本。而当用户再编辑这篇文章的时候,按需要的应该展示纯文本,但是却展示了带div p 等标签的内容。

封装一个方法,用于取标签间的纯文本

  • 通过axios发送get/post请求时,url:”/topics/update” ：后面不要带空格,否则报400错误信息!! ^^
